# Производительность без JavaScript (материалы к докладу на HolyJS)

Исследование и демо-артефакты к докладу об **ускорении веб-страниц без опоры на клиентский JavaScript**: транспорт, TLS, возможности HTTP, ввод-вывод на сервере, сжатие и облегчённая разметка. Порядок изложения совпадает с порядком папок в репозитории ниже.

### `00-tcp-fast-open`

**TCP Fast Open (TFO)** — настройка ядра и сервера (в т.ч. Nginx `fastopen=`), сравнение трёхэтапного рукопожатия с TFO, компактные примеры pcap и скриншоты. Примеры C++-сервера и клиента в `src/`.

### `01-tls-false-start`

**TLS False Start** (RFC 7918) — как клиент может отправлять прикладные данные до завершения полного рукопожатия, настройки по умолчанию в браузерах (в т.ч. Chrome, Firefox), взаимодействие с TFO и захваты трафика для обычного и false start сценариев.

### `02-tls-early-data`

**TLS 1.3 0-RTT / early data** — возобновление сессии и «ранние» прикладные данные, сдвоенные схемы потока, варианты «только сервер» и полного обмена, компактные pcap с TLS 1.3 early data.

### `03-quic-0-rtt`

**QUIC и 0-RTT** — обзор стека QUIC, сравнение 0-RTT и «обычного» первого полёта, включение по умолчанию в браузерах, небольшой пример на Go в `src/`.

### `04-cipher-performance`

**Стоимость криптоопераций для наборов шифров** — микробенчмарки (например, ED25519 и RSA: генерация ключа, подпись, проверка), таблицы в CSV/таблицах и визуализация для операций на эллиптических кривых и RSA.

### `05-ocsp`

**OCSP и stapling (прикрепление OCSP)** — проверка статуса сертификата без лишнего кругового обращения клиента к УЦ, когда сервер «приколачивает» свежий ответ OCSP; схемы и пример трафика со stapling.

### `06-file-read`

**Эффективная отдача статики** — блокирующее и неблокирующее чтение, Nginx + `sendfile`, прямой ввод-вывод, схемы в духе io_uring, заметки по `strace`. Демо на C++ и трассировщик системных вызовов в `src/`.

### `07-compressing-base`

**Сжатие ответов** (gzip, Brotli, zstd) — сжатие по словарю / динамическое, примеры вокруг идеи «сжатия базового» объёма полезной нагрузки, сводные таблицы.

### `08-early-hints`

**HTTP 103 Early Hints** — ранняя отправка подсказок для предзагрузки (например, `Link: rel=preload`) до финального ответа, примеры в виде заголовков/тегов и скриншоты.

### `09-speculative-rules`

**Speculation Rules API** — декларативная настройка prefetch/prerender из HTML, примеры скриншотов с работающими правилами.

### `10-service-worker`

**Service Worker** — моменты срабатывания и порядок относительно остального конвейера (в т.ч. диаграммы по времени); не тяжёлый JS-приложение — акцент на жизненном цикле и моменте выполнения работы.

### `11-different-media-sources`

**Форматы изображений и `<picture>` / `srcset`** — AVIF, WebP, JPEG и др. как способ сократить байты без нового JavaScript, скриншоты «до/после» по форматам.

### `12-clear-html-attributes`

**Вес HTML из-за атрибутов** — сокращение лишних `data-*` и похожей разметки на страницах, похожих на реальные; сравнение скриншотов «только нужное» и «очищенный» вес DOM.

### `98-unattached-assets`

**Общие материалы** — схемы потоков TLS 1.2/1.3, компактные примеры pcap и прочие файлы, которые относятся к разным разделам, но не к одной демо-папке.

### `99-after-all`

**Lighthouse / инструменты «после всех оптимизаций»** — скриншоты оценок или отчётов после применения полного набора техник (в т.ч. в Chrome, Firefox).

## `public/`

**Статическая HTML-страница фиксированного размера** (например, 32 КиБ) для повторяемых экспериментов с размером, сжатием или передачей без скриптов и картинок в самой странице.
